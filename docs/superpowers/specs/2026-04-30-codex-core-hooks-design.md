# Codex Support for core-hooks Plugin

## Goal

Add OpenAI Codex CLI compatibility to the core-hooks plugin as the first step toward multi-platform plugin support across the cc-toolkit marketplace.

## Context

Codex CLI shares a nearly identical plugin architecture with Claude Code:
- Same hook events (PreToolUse, PostToolUse, Stop, SessionStart)
- Same hooks.json format
- Same tool name (`Bash`) in hook matchers
- Same `${CLAUDE_PLUGIN_ROOT}` env var (backward compat)
- Separate manifest directory (`.codex-plugin/` vs `.claude-plugin/`)
- Separate marketplace path (`.agents/plugins/marketplace.json` vs `.claude-plugin/marketplace.json`)

Codex does not support these Claude Code hook events: `Notification`, `SubagentStop`, `SessionEnd`.

## Design

### Repo-Level Changes

**1. Codex marketplace file** at `.agents/plugins/marketplace.json`:

```json
{
  "name": "cc-toolkit",
  "plugins": [
    {
      "name": "core-hooks",
      "source": {
        "source": "local",
        "path": "./core-hooks"
      },
      "policy": {
        "installation": "AVAILABLE",
        "authentication": "ON_INSTALL"
      },
      "category": "Developer Tools"
    }
  ]
}
```

**2. Instruction file symlink**: `ln -sfn CLAUDE.md AGENTS.md`

No changes to existing `.claude-plugin/marketplace.json`.

### core-hooks Plugin Changes

**3. Add `.codex-plugin/plugin.json`**:

```json
{
  "name": "core-hooks",
  "description": "Core safety and workflow hooks",
  "version": "1.0.6",
  "hooks": "./hooks/hooks.codex.json"
}
```

**4. Add `hooks/hooks.codex.json`**:

Same structure as `hooks/hooks.json` but excluding events unsupported by Codex:
- Keep: `PreToolUse`, `PostToolUse`, `Stop`
- Drop: `Notification`, `SubagentStop`, `SessionEnd`

All hook commands and matchers remain identical since Codex uses the same tool names (`Bash`, `Write`, `Edit`, etc.) and the same `${CLAUDE_PLUGIN_ROOT}` variable.

**5. Expand attribution patterns in `pre_git_hook.py`**:

Add Codex/OpenAI attribution patterns alongside existing Claude patterns:

```python
claude_patterns = [
    # Claude Code
    r'Generated with \[Claude Code\]',
    r'claude\.ai/code',
    r'noreply@anthropic\.com',
    r'Generated with',
    r'Claude Code',
    # Codex / OpenAI
    r'Generated by Codex',
    r'Generated with Codex',
    r'noreply@openai\.com',
    r'openai\.com/codex',
    r'Codex CLI',
]
```

This benefits both platforms вЂ” blocks AI attribution regardless of which tool runs the hook.

### Unchanged Files

- `.claude-plugin/marketplace.json` вЂ” untouched
- `core-hooks/.claude-plugin/plugin.json` вЂ” untouched
- `hooks/hooks.json` вЂ” untouched
- `safety_guard.py` вЂ” `tool_name == 'Bash'` already works in Codex
- `post_tool_use.py` вЂ” no tool name checks, fully portable
- `system_notification.py` вЂ” no tool name checks, fully portable

## Future Expansion

Once core-hooks is validated, the same pattern applies to other plugins:
- Add `.codex-plugin/plugin.json` per plugin
- Add plugin entry to `.agents/plugins/marketplace.json`
- Skills (SKILL.md) work as-is on both platforms
- Skills with Claude Code-specific tool references get a dual-file variant (SKILL.codex.md) when needed
