# Task-loop — project parameters

This file holds **this project's parameters** for the task-loop worker. The worker's **complete
cycle and all general rules live in the `cycle-worker` agent contract** (the agent's system
instructions) — not here — so they update centrally with the plugin and never drift between
projects. A worker reads this file for the project-specific values its contract refers to, and
reads `docs/task-loop/directions.md` (human steering, highest priority) first.

It is generated by the `create-cycle` skill; the `{{...}}` placeholders below are filled with
this project's specifics.

## Where the cycle and rules live

The operating principles, the step-by-step cycle (recover → confirm → set up worktree → rubric →
spec/plan → TDD → verify → reconcile → doc-update → open-PR → request-merge → record), the
recovery-comment protocol, the control-protocol events, and the stop conditions are **all in the
`cycle-worker` agent contract**. This file only supplies the parameters above; `directions.md`
supplies steering. To read the cycle directly (e.g. to run a task by hand without the agent
contract), see `task-loop/agents/cycle-worker.md` in the plugin.
